1. What is Delta Live Tables (DLT)?

Delta Live Tables is a declarative ETL framework in Databricks used to build reliable, scalable, and maintainable data pipelines with built-in data quality, monitoring, and orchestration.

2. Why use DLT instead of ADF + Databricks?

DLT simplifies pipeline development by:

reducing boilerplate code
providing built-in data quality checks
handling dependency management automatically
offering monitoring and lineage out of the box

3. What are the core components of DLT?
Tables (@dlt.table)
Views (@dlt.view)
Expectations (data quality rules)
Pipelines

4. Difference between @dlt.table and @dlt.view
Feature	Table	View
Stored	✅ Yes	❌ No
Materialized	✅	❌
Use case	Final/Intermediate data	Transformation logic

5. What are Expectations in DLT?

👉 Data quality rules

import dlt

@dlt.table
@dlt.expect("valid_age", "age > 0")
def users():
    return spark.read.json("path")
6. Types of Expectations

expect_or_drop → drop bad records
expect_or_fail → fail pipeline

from pyspark.sql import SparkSession
from pyspark.sql.functions import explode, split, col

# Create Spark session
spark = SparkSession.builder.appName("SplitExplodeExample").getOrCreate()

# Input data
data = [
    ('alica', 'Badminton,Tennis'),
    ('bob', 'Cricket,Badminton')
]

schema = ['name', 'sports']

# Create DataFrame
df = spark.createDataFrame(data, schema)

# Show original data
print("Original Data:")
df.show()

# Transform: split + explode
result_df = df.select(
    col("name"),
    explode(split(col("sports"), ",")).alias("sport")
)

# Show result

result_df.show()
